Summary

I'm proud to present you the first release of DragMoveDrop project in collaboration with Pleyo

This is the first included drag and drop for GWT. It is also the first simple and optimized Drag and Drop. Tested on every Operating system, on every browsers, and build with a small computer (1Ghz and 512Mo Ram). His unique system of detecting allows you to increase performance when you are moving a widget.

That is not all, i'm preparing many examples of use ; as 2000 Widgets in a page, and keep a very fluid move on yours Widget, create and drag you own list, and many others behaviors (block Widget into a panel, copy a Widget on Drag, etc...)

To install DragMoveDrop, please look at gwt : wiki

Supported Drag && Drop Widgets (directly) :

  • Label.
  • Image.
  • Focus Panel.

(Every others Widget are indirectly droppable or draggable. Everything is explain on the Getting started guide).

4 new methods per Widget :

addDragListener(DragListener);
addDropListener(DropListener);
removeDragListener(DragListener);
removeDropListener(DropListener);

The methods handle 11 listeners :

DragListener DropListener
onDragStart(Widget sender, int x, in y) onDropEnter(Widget sender, Widget source)
onDragMouseMoved(Widget sender, int x, in y) onDropOver(Widget sender, Widget source)
onDragEnd(Widget sender, int x, int y) onDropExit(Widget sender, Widget source)
onDragEnter(Widget sender, Widget target) onDrop(Widget sender, Widget source)
onDragOver(Widget sender, Widget target)
onDragExit(Widget sender, Widget target)
onDragDropEnd(Widget sender, Widget target)

Control your Drag

The Drag Move Drop Project include the possibility to entirely control your drag :

  • Stop a move,
  • Sets a Draggable Widget, which move on the click of an other,
  • Sets Your detection padding,
  • Control the fire of the listeners,
  • Control the Drop Map, and disable some target,
  • Update the position of the drop target when you want to.
  • ...
